Background

This source code auditing platform was designed for security-conscious clients in the financial industry. It allows users to upload, scan, and generate reports on their source code. Future development plans include adding vulnerability analysis features. I was responsible for both the UX and UI design of this project.

Challenges

  • Permissions: Users need to see different projects based on their assigned projects and groups.
  • Progress Status: Users need to clearly understand the scan stage for each project when multiple projects are running.
  • Security Considerations: High-level security considerations were incorporated throughout the design process.

Solutions

  • Automated Scanning: Users can upload eligible files, and the system will run the scan in the background. Users don't need to stop their workflow for the scan.
  • Visualized Progress: In addition to listing scan progress in a table, the project detail page uses a more graphical flowchart to show the current scan stage, enabling quick understanding of the scan status.
  • Flexible Group Management: The group structure allows system administrators to assign users to different groups and specify the projects visible to each group, maintaining flexible permissions for all members.

Benefits

  • Enhanced Security: Users can independently upload, scan, and generate reports for their code, improving application and system security and saving development time.
  • Quick Problem Identification: The flowchart helps users quickly identify the stage at which problems occur during scanning, allowing them to seek assistance with specific information.
  • Flexible Permission Management: The group design provides system administrators with high flexibility in managing projects and accounts and meets security requirements for permission segregation.

Takeaways

  • Catering to Different User Levels: This design used multi-select menus without checkboxes. While keyboard shortcuts like Ctrl, Shift, and Ctrl+A were included, this feature should be considered a shortcut for experienced users. Without checkboxes, new users might not realize the multi-select option.
  • RWD with Maximum Flexibility: In the past, RWD designs typically created separate layouts for different device sizes. However, based on engineer suggestions, this project incorporated max/min width into each component to handle RWD when users resize their windows.
  • Real-time vs. Submit-based Input Validation: In programming, input validation can be done either during input or after the submit button is clicked. Designers need to consider this aspect and clearly indicate the display timing during user flow (or wireframe) creation.
