Central monitoring system purposal

Web Application

This proposal outlines a central monitoring system that integrates observation and alerting capabilities. It enables the customization of alert event types and triggers notifications via email, SMS, or other channels to designated response personnel. This system aims to achieve early detection and rapid resolution of hazardous events.

Sitemap

In the requirements document, the client has outlined the following needs:

  • A centralized information page that also allows for detailed viewing of individual sources.
  • Real-time notification functionality to alert relevant response units upon the occurrence of emergency events.
  • Provision of tailored information for command center personnel based on the type of emergency event. Ideally, command center personnel should have the flexibility to customize the content for optimal situational awareness.
  • Implementation of an RBAC (Role-Based Access Control) system to manage user permissions.

In response to these client requirements, I have created sitemaps for both the front-end and back-end to ensure that all functionalities and content are included in the development plan.

In addition to the client's explicit requirements, I have also incorporated the following enhancements:

  • A contact management feature to streamline system administration.
  • Considerations for command center hardware, such as an uninterruptible power supply (UPS) and siren notifications.

Overall, the proposed solution not only addresses the client's stated requirements but also proactively enhances the system's functionality and robustness, ensuring a comprehensive and reliable command center solution.

Wireflow

Through my involvement in various projects, I have gradually accumulated my own Component Library and Design Kit. This has enabled me to create wireframes with a higher degree of completion more quickly. By combining text, arrows, and flow markers, I can generate wireflows that allow collaborators and developers to understand the progress of the user flow and facilitate discussions.

A unique aspect of this design is the requirement for different screen sizes. This command center will be used on a video wall, so in addition to the standard FHD screen size, another requirement is for a 3840x1920 screen size.

When designing the UX, I followed the approach of designing for small screens first and then scaling up to larger screens. This ensures that essential information is presented consistently across all screen sizes.

UI Design

To establish a more organized Design Kit and ensure color consistency across the interface, I utilized Figma's Local Variable feature to define Color Variables. As much as possible, all components employ predefined colors. However, since gradient colors cannot be stored in Local Variables, they must be implemented using the Library.

After discussions with the UI designer, we decided to utilize transparency and background blur to enhance the technological feel of the products. We also employed gradient effects extensively, not only to enhance the aesthetics of the charts but also to visually represent the strength and categorization of numerical values.

Takeaways

  • Due to the large volume of information involved, effectively organizing and structuring the information was a crucial aspect of this project.
  • Designing for different screen sizes posed a unique challenge that required careful consideration.
  • The abundance of data presented necessitated the implementation of extensive filtering options within the tables, for which we conducted thorough research to ensure an intuitive and user-friendly design.
