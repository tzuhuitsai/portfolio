Data Visualization SaaS Editor

SaaS Product

To address market demand, we aimed to create a dedicated tool to handle the extensive requirements for situation room setups. This tool would allow for clear role delineation and collaboration among various project members. Additionally, we planned to leverage cloud technology to develop this product as a SaaS offering.

Research

I conducted a thorough analysis of current data visualization and BI products available on the market, focusing on their strengths, weaknesses, features, and market positioning. Most products either do not offer or only provide basic 3D and GIS functionalities but excel in data analysis capabilities. Generally, these products have a wide variety of chart types. Given our productâ€™s focus on situation rooms, we identified the following features as our key highlights:

  • API Integration
  • GIS functions
  • 3D functions
  • Triggered events

Target Audience

  • Corporate Departments: These users have clear role delineations and require ongoing maintenance of their situation rooms. They need their own domains and must adhere to branding guidelines and shared assets.
  • Studios: With more flexible role boundaries, studios need adaptable permission settings. Each project has its own client, necessitating functionality for external project sharing.
  • Freelancers: Freelancers may not possess all the necessary skills. Supportive tools in specific areas would greatly enhance their work efficiency.

Challenges

  • Designing for diverse use cases requires offering users significant freedom.
  • To facilitate smooth collaboration among different roles, we needed to design role-specific functionalities and permissions, along with robust version control.
  • Handling multiple data sources and ensuring data security were crucial.
  • The 3D and 2D components in the situation room must respond interactively to data changes and user interactions, such as displaying pop-ups or changing component appearances.
  • Given the various possibilities, the interface needed to be clear to prevent user confusion.

Solutions

  • Role-Based Access Control (RBAC): We implemented a system with workspace owners, administrators, designers, and developers. This setup ensures strict control over project visibility, data sources, and assets, managed by the workspace administrators.
  • Version Control: Unlike typical cloud editors, we introduced a testing environment. Users can complete testing in this environment before pushing versions to the production environment. This, combined with version control, aids corporate departments in iterating and refining their dashboards.
  • User-Specific Design: We created various visual styling tools, enabling users without design expertise to create visually appealing interfaces that adhere to design standards. An intuitive process for creating interactive components allows non-programmers to set up complex interactions. Additionally, DNS settings enable enterprise users to publish projects using their custom domains.

Benefits

This tool provides tailored solutions for individual users, studios, and corporate departments. Designers can complete projects independently, while clear collaboration features support team efforts. Engineers working solo also benefit from visual enhancement tools, saving time on color adjustments and ensuring visual consistency. Adding 3D elements to dashboards not only enhances visual appeal but also helps users better understand the data being observed.

Takeaways

This has been the most complex project I've worked on so far. As the sole UI/UX Designer, I had significant freedom and decision-making power to implement my ideas on user experience and product design. However, this also meant considering numerous details. Designing a SaaS product requires balancing user needs with technical constraints.
